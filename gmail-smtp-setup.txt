Gmail / Google Workspace SMTP Setup — gehaltsdeckel.jetzt
==========================================================

GOAL: Use Google as SMTP relay, sending FROM noreply@gehaltsdeckel.jetzt
      without the "via gmail.com" annotation in recipients' email clients.

SMTP SERVER (same for both options below)
------------------------------------------
  Host:     smtp.gmail.com
  Port:     587
  Security: STARTTLS


OPTION A — Google Workspace (recommended, cleaner)
----------------------------------------------------
If your MX records already point to Google (Google Workspace / Google MX),
you can create a real noreply@gehaltsdeckel.jetzt account in Workspace and
use it directly — no "via gmail.com" annotation, no extra steps.

  1. In Google Admin (admin.google.com) create the user noreply@gehaltsdeckel.jetzt
     (or use an existing Workspace account you want to send from)
  2. Enable 2-Step Verification on that account
  3. Create an App Password for it (see Step 1 below for how)
  4. Set in .env:
       GMAIL_USER=noreply@gehaltsdeckel.jetzt
       GMAIL_APP_PASSWORD=xxxx xxxx xxxx xxxx


OPTION B — Personal Gmail with "Send mail as"
----------------------------------------------
If you are using a personal @gmail.com account and want it to send as
noreply@gehaltsdeckel.jetzt, follow Steps 1–3 below.


STEP 1 — Create a Gmail App Password
--------------------------------------
1. Go to the Gmail account you want to send from (e.g. yourname@gmail.com)
2. Enable 2-Step Verification if not already on:
   https://myaccount.google.com/security
3. Go to App Passwords:
   https://myaccount.google.com/apppasswords
4. Create a new App Password → name it "gehaltsdeckel-smtp"
5. Copy the 16-character password (format: xxxx xxxx xxxx xxxx)


STEP 2 — Add credentials to .env
-----------------------------------
Add these two lines to your .env file on the server:

  # Google Workspace account:
  GMAIL_USER=noreply@gehaltsdeckel.jetzt
  # OR personal Gmail account:
  # GMAIL_USER=yourname@gmail.com

  GMAIL_APP_PASSWORD=xxxx xxxx xxxx xxxx


STEP 3 — Remove "via gmail.com" annotation (optional but recommended)
------------------------------------------------------------------------
This step makes Gmail send natively as noreply@gehaltsdeckel.jetzt.

3a. noreply@gehaltsdeckel.jetzt must be able to receive email so Gmail can
    send the verification code to it. Set up a forward to your Gmail:

    In your DNS/mail provider (wherever the MX records for gehaltsdeckel.jetzt
    are managed), create an email forward:

      noreply@gehaltsdeckel.jetzt  →  gehaltsdeckel.jetzt@gmail.com

    Provider-specific instructions:
    - Hetzner DNS Console: Domains → gehaltsdeckel.jetzt → Email Routing
    - Cloudflare: Email → Email Routing → Custom addresses → Add address
    - IONOS / 1&1:  Email → Forwarding → Add forwarding address
    - Netcup:       Domains → gehaltsdeckel.jetzt → Postfach/Weiterleitung

    Note: email forwarding requires an MX record pointing to your provider's
    mail servers. If you are using Cloudflare Email Routing, they add those
    automatically. Otherwise check your provider's docs for the required MX.

    Once the forward is active, proceed to step 3b — the confirmation code
    Gmail sends to noreply@gehaltsdeckel.jetzt will arrive in
    gehaltsdeckel.jetzt@gmail.com.

3b. In Gmail:
    Settings (gear) → See all settings → Accounts and Import
    → "Send mail as" section → Add another email address

3c. Enter:
    Name: Gehaltsdeckel Initiative
    Email: noreply@gehaltsdeckel.jetzt
    Uncheck "Treat as an alias"
    → click "Next Step"

3d. IMPORTANT — SMTP server dialog:
    Gmail shows this because you unchecked "Treat as an alias".
    Enter Gmail's own SMTP here:

      SMTP Server: smtp.gmail.com
      Port:        587
      Username:    yourname@gmail.com   ← your full Gmail address
      Password:    your 16-char App Password (from Step 1)
      Security:    TLS (recommended)

    → click "Add Account"

3e. Gmail sends a confirmation code to noreply@gehaltsdeckel.jetzt.
    The forwarding rule from 3a delivers it to your Gmail inbox.
    Enter the code to complete verification.

3f. Set noreply@gehaltsdeckel.jetzt as default sender in Gmail settings
    if desired.


STEP 4 — Deploy
-----------------
On the server, redeploy with:

  docker compose pull
  docker compose up -d --build

Haraka has been removed. The app now connects directly to smtp.gmail.com:587.


STEP 5 — Verify
-----------------
Watch app logs for nodemailer errors:

  docker compose logs -f app

A successful send looks like:
  nodemailer: message sent, messageId: <...@gmail.com>

If you see "Invalid login" → check GMAIL_USER and GMAIL_APP_PASSWORD in .env
If you see "Username and Password not accepted" → ensure 2-Step Verification
  is enabled and you are using an App Password (not your regular password)


NOTES
------
- Gmail free tier limit: 500 emails/day (Google Workspace: 2000/day)
- Attachment size limit: 25 MB per message
- The Haraka container and config are still in the repo but no longer used.
  They can be deleted once this is confirmed working.
